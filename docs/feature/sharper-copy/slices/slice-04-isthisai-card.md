# Slice 04: isthisai project card reframe (all languages)

## Goal
The isthisai card makes the reader think "I want to see this" — not "I understand what this is."

## IN scope
- Rewrite `isthisaiDescription` in EN, RO, HU
- The description should frame the challenge, not define the product

## OUT scope
- isthisaiCta (already good: "Play isthisai")
- isthisaiStatus (addressed in S6)
- Layout changes

## Learning hypothesis
A challenge-framed description ("Can you tell if this was made by a human?") increases click-through vs. a definition-framed one ("A game about telling the difference"). Disproved if analytics show no change or decrease in isthisai CTA clicks.

## Acceptance criteria
- The description makes the reader want to click, not just understand
- It preserves the core information (images/videos, human vs. machine judgment)
- It does NOT add enthusiasm or hype — sharper framing, not louder volume
- RO and HU preserve the challenge framing

## Dependencies
- S1 (hero sets the tone; project cards should match the hero's specificity)

## Effort estimate
~1 hour (3 languages)
